# 02 — 可替換資料總覽

本程式最重要的設計決策之一：**所有字碼表、字型、九宮格圖片都可以被使用者替換，
完全不需要重新編譯程式。**

這篇說明替換機制怎麼運作、哪些檔案可替換、替換後如何驗證、遇到問題怎麼偵錯。

---

## 一、為什麼能替換？

程式的所有資源都內嵌在 `.exe` 裡（以 WPF 的 Resource 方式，執行時從 Pack URI 載入）。
但啟動時，它會額外掃描執行檔同目錄下的兩個子資料夾：

- `Data\` ← 字碼表、字型
- `UI\` ← 九宮格圖片

**如果這兩個資料夾裡有同名檔案，就用你的、蓋掉內嵌的。**

這個機制叫「Resource Override」。它的好處：

1. **不需要 recompile**：使用者沒有開發環境也能自訂
2. **檔案完全獨立**：你可以 git 管理你自己的資料集，和程式本體互不干擾
3. **可攜**：整個資料夾複製到另一台電腦仍然有效
4. **安全 fallback**：如果你的檔案有問題（無法讀、格式錯），程式會**悄悄退回**
    使用內嵌資源，不會崩潰
5. **清楚的視覺標記**：一旦偵測到有檔案被覆蓋，標準介面左下角的 icon 會從
    變成**紅色**，提醒你目前跑的是自訂資源

## 二、檔案結構

把執行檔放在任意資料夾，下面是完整可用的結構：

```
D:\Portable\G9input\
├── G9inputDemo.exe       ← 必須，程式本體
├── settings.conf         ← 選用，USB 模式啟用時自動產生
├── Data\                 ← 選用，字庫覆蓋資料夾
│   ├── char_map.txt      ← 九宮格字碼表（核心必換項）
│   ├── rel_map.txt       ← 關聯字表
│   ├── cj5_map.txt       ← 倉頡碼表
│   ├── b5_map.txt        ← 大易碼表
│   ├── homo_map.txt      ← 粵語同音字表
│   ├── b5_uni.txt        ← Big5↔Unicode 轉換表
│   ├── t2s.txt           ← 繁簡轉換單字表
│   └── font.ttf          ← 應用程式字型
└── UI\                   ← 選用，UI 圖片覆蓋資料夾
    ├── 0s.png            ← 首頁背景
    ├── 0n.png            ← 2 碼減弱背景
    ├── 0c.png            ← 關聯預覽背景
    ├── 0f.png            ← 空頁背景（未有字）
    ├── 1.png~9.png       ← 第一碼為 1~9 時的大背景圖
    ├── 0-1.png~0-9.png   ← 迷你 UI 用：Home 狀態的 9 個單格提示
    └── 1-1.png~9-9.png   ← 迷你 UI 用：各首碼下 9 個單格提示
```

## 三、載入優先順序

程式啟動時的資源讀取順序：

```
┌─────────────────────────────────────┐
│ 1. 檢查 exe 同目錄的 Data\<檔名>       │
│    有 → 從檔案系統讀取                 │
│    無 → 下一步                        │
│                                     │
│ 2. 檢查 exe 內嵌 pack URI 資源         │
│    有 → 從 exe 資源讀取                │
│    無 → 拋出錯誤（不應發生，除非 exe 損壞）│
└─────────────────────────────────────┘
```

UI\ 資料夾的判定和 Data\ 完全平行。每個檔案**獨立**判定，
這意味著你可以**只換其中一個**——例如只換 `char_map.txt` 但保留所有內嵌圖片，
或者只換一張 `1.png` 但保留其他圖片。

## 四、完整檔案清單

### 4-1. 資料表（放 `Data\`）

| 檔名 | 說明 | 必要性 | 完整文件 |
|---|---|---|---|
| `char_map.txt` | **九宮格字碼對照表**，決定打什麼碼出什麼字 | 核心 | [字庫_char_map.md](字庫_char_map.md) |
| `rel_map.txt` | 關聯字表，每字對應 9 個「下一字」候選 | 常用 | [字庫_rel_map.md](字庫_rel_map.md) |
| `cj5_map.txt` | 倉頡五代碼表，給「轉倉」輸出用 | 少用 | [字庫_cj5_map.md](字庫_cj5_map.md) |
| `b5_map.txt` | 大易碼表，給「轉易」輸出用 | 少用 | [字庫_b5_map.md](字庫_b5_map.md) |
| `homo_map.txt` | 粵語同音字表，給右鍵同音字用 | 選用 | [字庫_homo_map.md](字庫_homo_map.md) |
| `b5_uni.txt` | Big5↔Unicode 對照表，HKSCS PUA 字元轉換 | 選用 | [字庫_b5_uni.md](字庫_b5_uni.md) |
| `t2s.txt` | 繁簡轉換表（繁→簡，單字對照） | 選用 | [字庫_t2s.md](字庫_t2s.md) |

### 4-2. 字型（放 `Data\`）

| 檔名 | 說明 |
|---|---|
| `font.ttf` | 整個 UI 使用的字型。程式啟動時載入其中第一個字型家族（family）。 |

字型會影響：
- 九宮格大候選字的繪製
- 關聯字小字
- 狀態列「標點 / 姓氏 / 確定 / 下一頁 / 取消」文字
- 底部字碼顯示區的數字

建議字型要同時涵蓋：
- 基本漢字（BMP 區，4E00-9FFF）
- 擴展 A / B / C 區（若你的碼表會用到冷僻字）
- HKSCS 的 PUA 區（U+E000~U+F8FF，若你有香港擴展字）
- 基本拉丁字母與標點
- 至少繁體字形

DEMO 版內建的字型涵蓋常用繁體字。若你的字庫包含大量冷僻字（例如「𠀀」這種擴展區 B 字元），
**必須**換一份支援這些字的字型，否則 UI 會顯示豆腐方塊。

### 4-3. UI 圖片（放 `UI\`）

| 檔名 | 尺寸 | 用途 |
|---|---|---|
| `0s.png` | 204×206 | 標準 UI / Home 狀態的 3×3 背景圖（9 種基本筆型） |
| `0n.png` | 204×206 | 標準 UI / SecondDigit 狀態（2 碼淡化） |
| `0c.png` | 204×206 | 標準 UI / AssociatePreview 狀態（關聯字背景） |
| `0f.png` | 204×206 | 標準 UI / Selecting 狀態下「該頁沒字」的空頁 |
| `1.png`~`9.png` | 204×206 | 標準 UI / FirstDigit 狀態（第一碼為 1~9 時的提示圖） |
| `0-1.png`~`0-9.png` | 30×30 | 迷你 UI / Home 狀態的 9 個單格提示（對應 9 種筆型） |
| `1-1.png`~`1-9.png` | 30×30 | 迷你 UI / FirstDigit = 1 狀態的 9 個單格提示 |
| `2-1.png`~`2-9.png` | 30×30 | 迷你 UI / FirstDigit = 2 狀態的 9 個單格提示 |
| ...（以此類推，到 9-9.png） | 30×30 | |

**總計**：標準 UI 用的大圖 13 張（0s/0n/0c/0f + 1~9），迷你 UI 用的小圖 90 張
（0-1~0-9，1-1~1-9，…，9-1~9-9）。

完整的尺寸、座標、字佔區域、繪製規範見
[`03_UI圖片設計指南.md`](03_UI圖片設計指南.md)。

### 4-4. `settings.conf`（選用，USB 模式專用）

這個檔不是「覆蓋檔」，是程式自己管理的可攜設定檔。USB 模式啟用時程式會把設定寫到這裡，
關閉時會刪除。內容為簡單的 `key=value` 純文字格式。

## 五、替換流程範例

### 範例 1：只換首頁背景圖

假設你想把九宮格的基本筆型圖案換成自己的設計：

1. 準備一張 204×206 的 PNG，命名為 `0s.png`
2. 在 exe 同目錄下建立 `UI` 資料夾
3. 把 PNG 放進去：`UI\0s.png`
4. 啟動程式（如果已開啟需重啟）
5. 首頁出現的圖變成你的設計。左下角 icon 變紅色 = 確認有覆蓋生效。

如果圖片格式錯誤（不是 PNG、尺寸差很多等），程式會忽略它、繼續用內嵌版本，
icon 不會變紅。

### 範例 2：整套換字庫和圖

1. 準備一整組新的 `Data\*.txt` 和 `UI\*.png`（或部分）
2. 確保格式符合後面各文件的規範
3. 放到 exe 同目錄的 `Data\` 和 `UI\`
4. 啟動程式
5. 開啟「關於」視窗 → 應該看到詳細的覆蓋清單，列出每一個被覆蓋的檔案
6. 實際操作測試一下，確認行為符合預期

### 範例 3：只替換 `char_map.txt` 但保留其他

這是最常見的使用情境——重新分派筆型、或擴充／縮減字庫。

1. 在 exe 同目錄建立 `Data\` 資料夾
2. 只放一個 `char_map.txt` 進去
3. 其他檔案（`rel_map.txt` 等）保持不存在，程式會用內嵌版本

注意：如果你只換 `char_map.txt`，但沒同步換 `rel_map.txt`，關聯字功能會變得亂：
例如你把「我」在新表裡拆成 `492`，送出後查 `rel_map.txt` 中「我」底下的關聯字仍是原本那組，
那組關聯字的拆碼在你新表裡可能完全不通。

要**徹底**改變輸入法體驗，以下 3 個檔通常要一起調整：

- `char_map.txt`
- `rel_map.txt`
- `0s.png`（首頁背景的筆型圖）與 `0-1.png`~`0-9.png`（迷你同款）

### 範例 4：擴充百家姓

百家姓的定義在 `char_map.txt` 的 `10,...`~`90,...` 這 9 行裡。假設你想在首碼為 5 的姓氏
清單裡加入一些港式罕見姓氏：

1. 開啟內建 `char_map.txt`（或直接從本文件的範例複製）
2. 找到 `50,孫陳張鄧陸司孔阮陶費紀樂陽...`
3. 在結尾加上你要的字：`50,孫陳張鄧陸...習終弘隆那乜巢紅甄鄺吳`
4. 另存到 `Data\char_map.txt`
5. 重啟程式

## 六、驗證替換有沒有生效

### 視覺快檢

1. **左下角 icon 顏色**：綠色（DEMO 原樣）／紫色（正式原樣） → 紅色（有覆蓋）
2. **「關於」視窗**：從托盤選單或視窗右鍵開啟，會列出所有被覆蓋的檔案的絕對路徑和
    檔案大小
3. **試打一個你改過的字**：如果新碼生效就 OK

### 進階：透過「關於」視窗的檢查清單

「關於」視窗中的「自訂資源」區塊會顯示類似：

```
字庫檔案
  ✓ char_map.txt         D:\Portable\G9input\Data\char_map.txt    37,156 bytes
  ○ rel_map.txt          （使用內嵌版本）
  ○ cj5_map.txt          （使用內嵌版本）
  ...

字型檔
  ○ font.ttf             （使用內嵌版本）

UI 圖片（3 個檔被覆蓋）
  ✓ 0s.png               D:\Portable\G9input\UI\0s.png            38,902 bytes
  ✓ 1.png                D:\Portable\G9input\UI\1.png             30,011 bytes
```

這裡的「✓」代表已被你的檔案覆蓋；「○」代表仍用內嵌版本。

## 七、偵錯：為什麼我的替換沒生效？

### 症狀：icon 沒變紅，檔案看起來放對了

可能原因：

1. **放錯位置**：`Data\` 資料夾必須在 **exe 同目錄**，不是 `My Documents` 或
    `AppData`
2. **檔名大小寫**：Windows 檔案系統對大小寫不敏感，但建議一律**全小寫**
3. **路徑太長**：Windows 預設限制 260 字元，把資料夾搬到較短路徑試試
4. **UAC 或防毒鎖定**：某些防毒軟體會阻擋程式讀取特定路徑，試試放在
    `C:\Users\你的使用者\Documents\G9input\`

### 症狀：icon 變紅，但行為沒變

- 確認你**關掉了舊的執行中程式**再開新的。程式不會熱重載。
- 確認你改對的是**對應的那張圖**——例如改了 `1.png` 但實際顯示的狀態是 Home，
    那就該改 `0s.png`。參考 [`01_輸入法基礎.md`](01_輸入法基礎.md) 的狀態機對照。

### 症狀：改了字庫後程式打字打一半會當機

可能是字庫格式問題。回 [`字庫_char_map.md`](字庫_char_map.md) 仔細核對每行的格式：

- UTF-8 編碼（**不要**是 Big5 或 GBK）
- 逗號是半形 `,`（**不要**中文全形 `，`）
- 每行一筆，不要有多餘的空白行
- 3 碼 key 必須是 ASCII 數字字元 `0`~`9`

把你的 `char_map.txt` 用 VS Code 開啟，右下角看編碼確認是 UTF-8；
如果是 UTF-8 with BOM 也 OK，程式會自動偵測並跳過 BOM。

### 症狀：改了圖片後某些狀態顯示黑色方塊或透明

- PNG 一定要是 **RGBA**（32-bit，有 alpha 通道）。
- 用 Photoshop 或 GIMP 存檔時選「PNG-24」或「PNG with transparency」
- 不要用 8-bit indexed PNG（雖然程式多半也能讀，但在某些 Windows 版本上會有色偏）

## 八、進階：自動化字庫建置

如果你要維護一份大型字庫（數千~數萬字），建議建立自動化流程：

```
ref_data.csv (你的原始資料，例如 Excel 匯出)
     │
     ▼ (Python / Ruby / Node 腳本)
char_map.txt + rel_map.txt + t2s.txt
     │
     ▼ (copy)
D:\Portable\G9input\Data\
```

做自動化時要注意：

1. **UTF-8 編碼**（無 BOM 或有 BOM 都可）
2. **Unix 或 Windows 換行符皆可**（程式用 `StreamReader.ReadLine()` 兩者都吃）
3. **字元正規化**：建議把所有漢字先跑一次 `NFC` 正規化
4. **去重**：同一個 key 出現兩次後者會覆蓋前者（對 `char_map.txt` 以外的表）

---

## 九、邊界與已知限制

1. **不支援 runtime reload**：修改檔案後必須重啟程式
2. **不支援加密資料庫**：所有字庫是純文字，任何人都能讀
3. **單檔案大小實務上限 ~50 MB**：雖然技術上無限，但過大檔案會讓啟動時間顯著增加。
    DEMO 內建的 `b5_map.txt` 約 130 KB、`cj5_map.txt` 約 100 KB、`font.ttf` 約 7 MB，
    這是典型規模。
4. **檔名固定**：不能換成自己的命名，例如 `char_map_v2.txt` 不會被讀取。
    如果你要版本管理，建議另外備份資料夾。
5. **子資料夾不遞迴掃描**：`Data\sub\char_map.txt` 不會被讀到。只掃描 `Data\` 一層。
