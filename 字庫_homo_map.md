# 字庫 — `homo_map.txt`

> 粵語同音字表。提供「右鍵查同音字」功能，以及作為「粵語九宮數字組（0~9）」
> 一鍵查出所有同音字的工具。

---

## 一、角色定位

`homo_map.txt` 回答兩個問題：

1. **「按了粵語數字 N（0~9），屬於 N 類的所有同音字是哪些？」**（用於「粵語數字模式」）
2. **「中文字 X 的粵語同音字有哪些？」**（用於「右鍵查同音字」）

兩種用法對應同一份表，只是 key 的類型不同。

### 1-1. 用法 1：右鍵查同音字

使用者在 Selecting 狀態下**右鍵點擊**任一有字的格子，程式以該字作為 key 查 `homo_map.txt`，
把查到的同音字作為新的候選列表替換顯示在九宮格裡：

```
使用者選字狀態，看到 9 個候選字
   ↓
右鍵點「伊」
   ↓
查 homo_map.txt["伊"] = "2姨擬E疑耳蛇豫誼而意易二衤..."
   ↓
九宮格換成顯示「2」「姨」「擬」「E」「疑」「耳」「蛇」「豫」「誼」
（狀態列提示「選同音字」）
```

### 1-2. 用法 2：粵語數字模式（按 `0`~`9` 當 key）

這不是一個獨立的輸入模式——它是用法 1 的特例。`homo_map.txt` 裡的 key 可以是 `0`~`9`
的單一數字字元，查詢時如果第一個字元剛好是這些數字，程式會把它們作為粵語數字的
「音群 key」。但實際觸發這個功能需要使用者操作路徑（本 DEMO 文件不詳述，因屬於進階使用）。

**沒有 `homo_map.txt` 的話**：右鍵查同音字功能無作用，右鍵點擊只會觸發預設行為（什麼也
不做）。

## 二、檔案格式

### 2-1. 整體結構

純文字檔案。每行一筆：

```
<key> <chars>
```

- `<key>` 是查詢用的 key，可以是：
  - 單一漢字（例如「伊」）
  - 單一數字字元（`0`~`9`）
  - 非中文單字元（例如英文字母，實務上很少）
- **分隔符是單一半形空白 `' '`（ASCII 0x20）**
- `<chars>` 是同音字串

### 2-2. 編碼與換行

- **編碼**：UTF-8
- **換行**：`\n` 或 `\r\n`
- **空白行**：略過

### 2-3. 關鍵：分隔符是空白，不是逗號！

和 `char_map.txt` 不同，這裡用**空白**作為分隔符：

```
char_map.txt:  110,華萬夢若著  ← 逗號
homo_map.txt:  伊 2姨擬E疑耳  ← 空白
```

原因是：同音字的 key 可以是任何字元（例如英文字母、數字），若用逗號作分隔符，key 本身
不能是逗號；而用空白作分隔符的限制較少（空白在 CJK 和 ASCII 中都不常出現在 key 中）。

### 2-4. 讀取邏輯

程式讀每行時：

1. 找第一個空白字元 ` ` 的位置 `i`
2. `<key>` = `line[0..i]`（可能是 1 個 grapheme，但理論上可接多 char）
3. `<chars>` = `line[i+1..]`

注意：

- 若一行**沒有空白**，該行被忽略（無效格式）
- 若 key 部分是 2+ 個字元（如「伊人」），程式會把整個 `"伊人"` 當 key——雖然查詢
    時不會用雙字 key，所以這行實質無效。

### 2-5. chars 中的格位對應

和 `char_map.txt` 一樣：

- 第 1 個字素 → numpad 1 的格（左下）
- 第 2 → 2（中下）
- …
- 第 9 → 9（右上）
- 第 10+ → 第 2 頁、第 3 頁…

### 2-6. 不支援 `*` 佔位符

和 `rel_map.txt` 一樣，不支援 `*`。如果你想跳過格位，只能填實際字元。

### 2-7. 重複 key

```
伊 2姨擬E
伊 你好嗎
```

第一筆保留，第二筆被忽略（`Dictionary.TryAdd`）。若你要更新，修改第一筆。

## 三、key 設計原則

### 3-1. 兩套 key 並存

`homo_map.txt` 實際是**兩個獨立概念**塞在一個檔案裡：

**A. 漢字 key**（主要功能）

```
伊 2姨擬E疑耳蛇豫誼而意易二衤移議痍咿綺𪘲頤宜怡薏食倚以邇...
涉 葉躡攝懾
唎 呢嚟哩喱洱離狸霾利漓梨嚟厲力裏吏里膩哩蒞釐𢃇罹脷厘...
```

這是「同音字查詢」的基礎——使用者右鍵點「伊」，跳出「姨、擬、E、疑…」等讀 `ji1` 的字群。

**B. 數字 key**（用法 2，僅供進階使用）

```
0 菱嶺擰拎令能苓楞零齡玲領呤另稜靚軨羚陵〇鈴伶棱凌爧炩乒愣聆靈檸
1 日逸有壹一溢軼
2 廿也呀吔伊姨擬E疑耳蛇豫誼而意易二衤移議痍咿綺𪘲頤宜怡薏食倚以邇台依肄椅儀噫異施貳義胰兒夷咦爾醫已飴衣
3 衫叁三叄
4 肆死四
5 五嗯午吾唔晤伍誤
6 鹿綠六錄淥麓陸轆碌氯睩戮祿
7 柒漆七𨳍
8 捌八
9 赳媾究笱韭柩嚿詬苟韮咎糾灸久舊狗玖疚鳩垢尻溝九夠枸㞗購構救
```

這 10 行把「0~9」對應到粵語數字讀音群：

- `0` → 粵語 `ling4` 讀音群（零、伶、菱…）
- `1` → `jat1` 讀音群（一、日、壹…）
- `2` → `ji6` 讀音群（二、義、異…）
- `3` → `saam1` 讀音群（三、衫…）
- `4` → `sei3` 讀音群（四、死、肆）
- `5` → `ng5` 讀音群（五、午、伍、嗯…）
- `6` → `luk6` 讀音群（六、祿、錄、陸…）
- `7` → `cat1` 讀音群（七、漆、柒…）
- `8` → `baat3` 讀音群（八、捌）
- `9` → `gau2` 讀音群（九、狗、久、舊、夠…）

這是粵語使用者特有的習慣——**打出粵語的諧音數字**去記憶詞彙。例如想打「你好」，
粵語讀 `nei5 hou2`，雖然這兩字和數字不同音，但同群字可以用數字 key 快速查到。

### 3-2. 漢字 key 的涵蓋範圍

最小必要集：`char_map.txt` 裡的所有常用字。

建議涵蓋至少常用 3,000 字，這樣使用者右鍵任何字都能看到同音字群。

### 3-3. 特殊 key

- 允許**非中文單字元** key（例如英文字母、HKSCS PUA 字元、Emoji）
- 但這些 key 在實際操作上很少被觸發（使用者通常不會去右鍵點英文字）
- 若你有對應的同音字群（例如 "E" 的粵語讀音是 `ji1`，與「伊」同群），可以加入

## 四、value 設計原則

### 4-1. 同音群的定義（粵語 Jyutping 為基準）

建議以 **粵拼（Jyutping）** 為分群基準，而不是用耳聽判斷。Jyutping 格式為
「聲母 + 韻母 + 聲調數字」，例如：

- 伊 = `ji1`
- 姨 = `ji4`

嚴格同音要求**聲母、韻母、聲調**都相同。但實務上，粵語的聲調在快速講話時會變調，
所以本表把**同聲母 + 同韻母**的字視為「同群」，聲調不同仍算作同群。

### 4-2. 排序原則

把**高頻字排前面**。例：

```
2 廿也呀吔伊姨擬E疑耳蛇豫誼而意易二衤移議痍咿...
```

「廿、也、呀」是高頻字，排前；「𪘲、脷、膩、匿」等是低頻或方言字，排後。

### 4-3. 收錄擴展字元

同音字表會自然涵蓋方言字，這些字常落在：

- **HKSCS PUA 區**（如「嘅」「喎」「嚟」「唔」）
- **CJK 擴展 B 以後**（如「𪘲」「𨳍」「𢃇」「㞗」）

程式支援這些字的顯示和輸出（配合 `b5_uni.txt` 作 PUA 轉換）。

## 五、載入行為與實作細節

### 5-1. 載入時機

啟動時載入，`FrozenDictionary<string, string>`。

### 5-2. 查詢時機

**Selecting 狀態下右鍵點擊有字的格子** → 取該格文字作為 key → 查 `homo_map.txt` →
若有結果，把結果作為新候選字列表 → 進入特殊「同音字選字」模式。

此模式下：

- 狀態列左邊顯示「選同音字」
- `+` / `-` 仍可翻頁
- 左鍵選字送出
- 再次右鍵會查**新字**的同音字（可層層深入）

### 5-3. 粵語數字 key 的觸發

程式本身**沒有**直接提供「按 0~9 輸入粵語數字」的輸入模式（這是用法 2，進階用法）。
但因為 `homo_map.txt` 裡 `0`~`9` 的 key 都有對應值，**理論上**可以：

- 在 Selecting 狀態右鍵點擊內容正好是 `0`~`9` 的候選格
- 程式會把該數字作為 key 查表，跳出對應的粵語數字同音群

實務上 `char_map.txt` 極少有候選字是純數字（因為九格輸入法不用於打數字），所以這個路徑
很少被觸發。

## 六、與其他字庫的關係

### 6-1. 與 `char_map.txt`

**理想**：`homo_map.txt` 的 key 子集於 `char_map.txt`（即所有 `char_map.txt` 裡的字都
有對應的同音字表）。

**現實**：同音字表可以**更大**——可以包含方言字（如「嘅」「喎」），即使這些字不在
`char_map.txt` 裡，右鍵同音查詢後的 value 字可能是這些方言字，仍然可以被選取輸出。

### 6-2. 與 `rel_map.txt`

完全平行獨立。`rel_map.txt` 是語義（下一個字），`homo_map.txt` 是語音（同音字）。
兩者可以分別查同一個字而得到不同的結果：

- `rel_map.txt["我"]` = "你這好到資就個能的"（下一個字）
- `homo_map.txt["我"]` = "...粵語 `ngo5` 同音字群..."

### 6-3. 與 `b5_uni.txt`

如果同音字表裡有 HKSCS PUA 字，使用者選中這些字後送出，會自動 HKSCS → Unicode 轉換。

## 八、作者的實務建議

### 8-1. 以 Jyutping 為基礎建表

如果你要從頭建表，推薦流程：

1. 取得「粵拼→字」的完整對照資料
   - 例如 [unihan.zip](https://www.unicode.org/Public/UNIDATA/Unihan.zip) 的 `kCantonese` 欄位
   - 或 [Rime 的 lib-cantonese](https://github.com/rime/rime-cantonese) 字典
2. 把字按粵拼分群（只看聲母+韻母，忽略聲調）
3. 每群排序（按字頻）
4. 產生 `homo_map.txt`，每個群的每個字都輸出一行，格式為：`<char> <同群其他字>`

### 8-2. 產生腳本範例

```python
from collections import defaultdict, Counter

# 假設 kCantonese.tsv 的格式是：char<TAB>jyutping
groups = defaultdict(list)
with open("kCantonese.tsv", encoding="utf-8") as f:
    for line in f:
        char, jp = line.rstrip().split("\t")
        # 取聲母+韻母（去掉末尾的聲調數字）
        base = ''.join(c for c in jp if not c.isdigit())
        groups[base].append(char)

# 也加入數字 key 的特殊組
special_mapping = {
    "0": "ling",  # 零 = ling4
    "1": "jat",
    "2": "ji",
    "3": "saam",
    "4": "sei",
    "5": "ng",
    "6": "luk",
    "7": "cat",
    "8": "baat",
    "9": "gau",
}

out_lines = []
# 先寫 0~9
for digit, base in special_mapping.items():
    group = groups.get(base, [])
    out_lines.append(f"{digit} {''.join(group)}")

# 再寫每個字對應自己同音群的其他字
for base, chars in groups.items():
    for i, char in enumerate(chars):
        others = [c for j, c in enumerate(chars) if j != i]
        if others:
            out_lines.append(f"{char} {''.join(others)}")

with open("homo_map.txt", "w", encoding="utf-8") as f:
    f.write("\n".join(out_lines) + "\n")
```

### 8-3. 減少冗餘

如果「伊」「姨」「椅」在同一群，那麼：

- `伊` 的 value 會包含「姨、椅…」
- `姨` 的 value 會包含「伊、椅…」
- `椅` 的 value 會包含「伊、姨…」

這些 value 很相似（差在少了自己）。這種重複是**無法避免**的——因為每個字要找自己的
同音字時，都會排除自己。

結果是 `homo_map.txt` 會變得比較大（DEMO 版 ~355 KB），但查詢是 O(1)，效能仍佳。

### 8-4. 方言字的取捨

要不要收「嘅、冇、唔、咁」這類粵語獨有的字？**強烈建議收**，因為這些是香港使用者
日常輸入的重要字，查同音字時使用者期待看到它們。

## 九、常見錯誤診斷

| 症狀 | 可能原因 |
|---|---|
| 右鍵沒反應 | 該字不在 `homo_map.txt` 裡，或 value 為空 |
| 右鍵跳出的字和預期差很多 | 同音字的分群標準不同，檢查你用的是聲母+韻母還是全拼 |
| 字順序很奇怪 | value 沒有按字頻排，檢查排序邏輯 |
| 某些字看起來是亂碼 | 該字是 HKSCS 或 CJK 擴展字元，但字型不支援 → 換字型 |
| 檔案過大影響啟動 | 355 KB 正常。若你的版本 >2 MB，檢查是否有大量重複內容 |

## 十、快速參考卡

```
格式：<key> <chars>（空白分隔）
編碼：UTF-8
分隔符：半形單一空白（0x20）
key：1 個字元（漢字、數字 0~9 或其他）
chars：同音字序列，順序 = 格位 1~9
數字 key：0~9 對應粵語數字音群
重複 key：首次生效
```
