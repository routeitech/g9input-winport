# 01 — 輸入法基礎

這一篇告訴你「怎麼用九格輸入法把一個字打出來」，以及支撐這件事的設計原理。
本篇不討論自訂資源，關於替換碼表／圖片的說明放在
[`02_可替換資料總覽.md`](02_可替換資料總覽.md) 與
[`03_UI圖片設計指南.md`](03_UI圖片設計指南.md)。

---

## 一、九宮格與九種基本筆型

鍵盤右側的數字鍵區（numpad）剛好是 3×3 網格，這套輸入法把它當成一個九宮格：

```
┌───┬───┬───┐
│ 7 │ 8 │ 9 │   ← 上排
├───┼───┼───┤
│ 4 │ 5 │ 6 │   ← 中排
├───┼───┼───┤
│ 1 │ 2 │ 3 │   ← 下排
└───┴───┴───┘
```

每個數字對應一種**基本筆型**。一個中文字在視覺上拆成至多 3 個最具代表性的筆劃／部件後，
每個筆劃歸類到 1~9 中最接近的類別。這 9 類的預設分派如下（可從程式的首頁背景圖 `0s.png`
清楚看到）：

| 鍵位 | 筆型 | 形狀表示 | 代表字例 |
|---|---|---|---|
| `7` | 橫 | ─ | 一、十、上、七 |
| `8` | 豎 | │ | 中、十、下、丨 |
| `9` | 撇 | ╱ | 人、川、仁、斜 |
| `4` | 點 | · | 心、冰、火、氵 |
| `5` | 折／乙 | Ƨ | 乙、已、己、鳥 |
| `6` | 方／口 | □ | 口、國、回、田 |
| `1` | 交叉（十字型） | + | 十、古、早、占 |
| `2` | 交叉（斜乂型） | × | 乂、父、又、交 |
| `3` | 勾／帽（上包覆） | ⌒ | 冂、內、同、冠 |

> 這 9 類分派**不是**硬寫在程式裡的。使用者若想重新分派，只要同步替換：
> ① 字碼表 `char_map.txt` 的碼→字對應，
> ② 首頁背景圖 `0s.png` 上顯示的筆型符號，
> ③ 單格提示圖 `0-1.png`~`0-9.png`。
> 可以自行建立一套全新的筆型系統。詳見
> [`03_UI圖片設計指南.md`](03_UI圖片設計指南.md)。

## 二、拆字原則

要輸入一個字，使用者需要把這個字拆成 1~3 個基本筆型，然後依順序按下對應的數字鍵。
拆字規則兩條：

### 規則 1 — 先左上、後右上

每個字至多取 3 個筆劃作為「碼」。取的順序是：

1. **第 1 碼**：字的最左上方筆劃（最先起筆的部位）
2. **第 2 碼**：字的最右上方筆劃（或與第 1 筆明顯分離的第二個主要部件）
3. **第 3 碼**：字的最右下方或最末筆劃

這個取向與倉頡「首、次、末」的思路類似，但判別標準是**視覺位置**而不是倉頡的字根集合，
一般人憑直覺比較容易上手。

### 規則 2 — 滿 3 碼自動選字

字碼最長為 3 碼。當：

- 1 碼無法唯一定位任何字（99.9% 的情況）
- 2 碼若該前綴下有唯一解會進入「確定」模式，但通常是 **2 碼就出候選池**
- 3 碼一定出候選池（9 個格子顯示符合 3 碼的字）

不滿 3 碼時可以點畫面上的「確定」狀態列提前查表。

## 三、五種輸入狀態（狀態機）

程式啟動後永遠處於下列五種狀態中的一種。理解狀態機是活用這套輸入法的關鍵。

```
          ┌──────────────────┐
          │      Home        │ ← 預設狀態，九宮格顯示筆型提示
          │     （首頁）       │   狀態列：[標點] [取消]
          └────────┬─────────┘
                   │ 按 1~9
                   ▼
          ┌──────────────────┐
          │   FirstDigit     │ ← 已輸入 1 碼
          │    （首碼）       │   九宮格顯示「該碼底下的 9 種次碼」
          │                  │   狀態列：[姓氏] [取消]
          └────────┬─────────┘
                   │ 按 1~9
                   ▼
          ┌──────────────────┐
          │   SecondDigit    │ ← 已輸入 2 碼
          │    （次碼）       │   九宮格淡化顯示，等待第 3 碼
          │                  │   狀態列：[確定] [取消]
          └────────┬─────────┘
                   │ 按 1~9 (第 3 碼) ───┐
                   │                    ▼
          ┌──────────────────┐  ┌──────────────────┐
          │   Selecting      │◄─┤ 點狀態列的「確定」 │
          │    （選字）       │  │（2 碼前綴搜尋）   │
          │                  │  └──────────────────┘
          │  九宮格顯示 9 個候選字
          │  按 1~9 → 送字
          │  按 +   → 下一頁
          │  按 -   → 上一頁
          │  右鍵某格 → 粵語同音字模式
          └────────┬─────────┘
                   │ 選字輸出
                   ▼
          ┌──────────────────┐
          │ AssociatePreview │ ← 剛送出一字，預覽下一字
          │    （關聯預覽）    │   九宮格回到筆型提示
          │                  │   每格左上角小字 = 關聯字候選
          │                  │   狀態列：[選字] [標點]
          └──────────────────┘
                   │ 按 0 或 點小字 → 輸出關聯字 → 留在本狀態
                   │ 按 1~9          → 起新字碼 → 回 FirstDigit
                   │ 按 . (Decimal)  → 回 Home
```

每個狀態底下的 UI 細節：

### 狀態 1：Home（首頁）

- **九宮格背景**：`0s.png`（大圖）或 `0-1.png`~`0-9.png`（小圖）
- **顯示內容**：9 種基本筆型的圖示
- **狀態列**：左「標點」（按 `0` 或點此進入標點模式）、右「取消」
- **左上角小字**：若上一次有送過字，9 個格子的左上角會顯示上一字的「關聯字」

### 狀態 2：FirstDigit（首碼）

- 使用者剛按了 `1`~`9` 的某一鍵
- **九宮格背景**：`1.png`~`9.png` 其中一張，顯示「第一碼為這個值時，各個次碼底下最常見的字是什麼」
- **狀態列**：左「姓氏」（按 `0` 進入百家姓）、右「取消」

### 狀態 3：SecondDigit（次碼）

- 使用者已經按了第 2 碼
- **九宮格背景**：`0n.png`（統一的淡化提示，n = narrowed），提示使用者「再打一碼會出字」
- **狀態列**：左「確定」（點此立即以 2 碼為前綴搜尋）、右「取消」
- **特殊**：第 2 碼為 `0` 會自動分岔到百家姓或擴充標點（見後）

### 狀態 4：Selecting（選字）

- **九宮格內容**：9 個大字的候選字，沒有背景圖
- 若該頁完全沒有字（前綴查無結果），顯示 `0f.png`（「未 有 字」）
- **狀態列**：左「下一頁」、右「取消」
- **鍵盤**：`1`~`9` 選字、`+` 下一頁、`-` 上一頁、`.` 取消回 Home

### 狀態 5：AssociatePreview（關聯字預覽）

- 使用者剛輸出一個字後進入本狀態（標準尺寸才顯示，迷你尺寸跳過本狀態）
- **九宮格背景**：`0c.png`（九種筆型的淡化版）
- **每格左上角小字**：上一個字的 9 個關聯字（來自 `rel_map.txt`）
- **狀態列**：左「選字」（其實就是上方小字任意點擊）、右「標點」

## 四、鍵盤對照表

### 數字鍵盤區（主要操作區）

| 按鍵 | 功能 |
|---|---|
| `1`~`9` | 選碼 / 選字 / 選關聯字 |
| `0` | 進入標點 / 擴充標點 / 關聯字選字 / 觸發姓氏（依目前狀態） |
| `.`（Decimal） | 取消 / 重置到 Home |
| `+`（Add） | 下一頁（選字模式中） |
| `-`（Subtract） | 上一頁（選字模式中） |
| `*`（Multiply） | 循環把視窗對齊到螢幕四個角落 |
| `/`（Divide） | **全域**顯示 / 隱藏輸入法視窗 |

### 其他可用的鍵盤區（5 種輸入裝置）

使用者可以依自己的鍵盤習慣啟用不同的輸入區塊。5 種可單獨或組合啟用：

| 裝置 | 說明 |
|---|---|
| **A. NumLock 開的數字鍵盤** | 走 Windows 全域熱鍵 `RegisterHotKey`。需 NumLock 開。 |
| **B. NumLock 關的數字鍵盤** | 走低階鍵盤鉤子攔截「掃描碼」，不論 NumLock 狀態皆可用 |
| **C. 文字鍵盤 數字列** | `1234567890-=` 這一整排對應 1~9 0 - + |
| **D. 文字鍵盤 左手** | QWE/ASD/ZXC/VB 映射到 789/456/123/0. |
| **E. 文字鍵盤 右手** | YUI/HJK/BNM/,. 映射到 789/456/123/0. |

D 和 E 因為在 `B` 鍵上衝突（D 的 `B` = Decimal，E 的 `B` = 1），UI 會強制互斥。

啟用低階鉤子後（B/C/D/E 中的任何一個），**被攔截的按鍵在輸入法視窗顯示時會完全失去原本的意義**
——例如啟用 D，記事本打不出 Q、W、E 這些字母。程式為此實作了**動態掛載**：
視窗一隱藏鉤子立即解除。常用的隱藏方法：按 `/`（numpad Divide）、按 ScrollLock（若有設定）、
雙擊系統托盤圖示、或點導覽列「隱藏」。

## 五、特殊模式

### 5-1. 標點模式

在 Home 狀態按 `0` → 顯示 9 個常用標點。常用標點的定義在 `char_map.txt` 的
`0,...` 那一行（詳見 [`字庫_char_map.md`](字庫_char_map.md)）。

### 5-2. 擴充標點

在標點模式下按 `+`（下一頁）或從 Home 按 `09` 兩個鍵 → 顯示擴充標點集。
擴充標點的定義在 `char_map.txt` 的 `09,...` 那一行。DEMO 版隨附約 370 個擴充標點。
可用 `+` / `-` 翻頁，每 9 字一頁。

### 5-3. 百家姓

在 Home 按 `1`~`9` 後再按 `0` → 進入該首碼底下的姓氏。例如：

- `1` → `0` → 趙、李、黃、胡、林…（來自 `char_map.txt` 的 `10,...`）
- `5` → `0` → 孫、陳、張、鄧…（來自 `50,...`）

這個設計是為了讓人名輸入更快，因為姓氏通常是出現頻率極高的有限集合。

### 5-4. 關聯字（選字後）

選了一個字後，系統會根據 `rel_map.txt` 查詢「這個字之後最常接哪些字」。
例如送出「我」之後，9 個關聯字可能是「你這好到資就個能的」。這些關聯字有兩種出現方式：

- **標準尺寸**：直接以小字顯示在 9 個格子的左上角，點小字即送字
- **迷你尺寸**：不顯示，但按 `0` 可進入「關聯字選字」模式，把 9 個關聯字放大到 9 格中選

### 5-5. 粵語同音字

在 Selecting 狀態中，**右鍵點擊**任何一個有字的格子 → 以該字為 key 查詢 `homo_map.txt`，
把同音字變成新的候選集進入選字。

`homo_map.txt` 也提供 9 個數字 key（`0`~`9`），各自對應一組「以該數字為粵語讀音相近的字」。
例如 `9` 對應「赳媾究笱韭柩嚿詬苟韮咎糾灸久舊狗…」——這些字的粵語讀音都是 `gau` 相關音。

## 六、輸出模式

選完字後，程式要把這個字「打」到目標文字欄位。有 4 種不同的輸出引擎：

| 模式 | 技術 | 說明 |
|---|---|---|
| **剪貼簿** | `Ctrl+V` | 最相容。把要輸出的字放進剪貼簿，送 `Ctrl+V`，再還原原剪貼簿內容 |
| **直出**（預設） | `KEYEVENTF_UNICODE` | 模擬鍵盤打入 Unicode 碼點，現代 Windows 應用都支援 |
| **轉倉** | 查 `cj5_map.txt` → 送 VK 虛擬鍵事件 + Space | 讓作業系統的倉頡 IME 組字 |
| **轉易** | 查 `b5_map.txt` → 送 VK 虛擬鍵事件 + Space | 讓作業系統的大易 IME 組字 |

轉倉／轉易這兩種是特殊用途：你在用系統倉頡或大易 IME 時，可以用九格輸入法來查字、
但讓最終的組字動作經過原本的 IME。這對於某些只認特定 IME 的老舊應用（例如部分舊版
Office、某些銀行系統的表單）特別實用。

### HKSCS PUA 自動轉換

香港擴展字元集（HKSCS）有一部分字落在 Unicode 的私人使用區（PUA，U+E000~U+F8FF），
這些字在現代 Unicode 裡可能有標準碼點對應。程式在送字前會自動：

1. 偵測字元是否在 PUA 區
2. 若是，以 Big5 編碼反查，再查 `b5_uni.txt` 找到對應的標準 Unicode
3. 用標準 Unicode 輸出

這讓使用者在候選字 UI 裡看到的紅色字（程式用紅色標示 PUA 字）都能正常送到不支援 HKSCS
的現代應用。

### 繁 ↔ 簡 輸出轉換

候選字 UI **永遠顯示繁體**。若使用者在托盤選單或導覽列切到「輸出簡體」，
送字前會用 `t2s.txt` 做單字對照轉換後才送出。

## 七、編碼過濾

有些老舊應用只支援 Big5 或 GB2312，如果送它不認得的字（例如 HKSCS 擴充字或 Unicode
輔助字面），會出現亂碼。程式提供編碼過濾模式：

| 模式 | 行為 |
|---|---|
| **UTF-8（顯示全部）** | 預設，不過濾任何字 |
| **Han（依繁簡）** | 繁體時以 Big5 編碼、簡體時以 GB2312 編碼，過濾送字失敗的候選 |

在 Han 模式下，候選字列表會**即時重新過濾**，只顯示那些能被目標編碼表示的字。

## 八、全域熱鍵

| 熱鍵 | 功能 |
|---|---|
| `numpad /`（Divide） | 顯示 / 隱藏輸入法視窗 |
| ScrollLock（若啟用） | 顯示 / 隱藏輸入法視窗的替代方案 |

`numpad /` 使用 Windows 的 `RegisterHotKey`，同一時間整個系統只能有一個程式註冊。
若該鍵被其他程式占用（例如某些會議軟體的靜音快速鍵），可在 `settings.conf` 裡把
`UseScrollLockInstead` 設為 `True`，改用 ScrollLock 偵測（以 DispatcherTimer 輪詢，
不會和其他程式衝突）。

## 九、視窗特性

輸入法視窗特別設計為：

- **不搶焦點**（`WS_EX_NOACTIVATE`）：點擊視窗本體不會把焦點從目標文字欄位搶走
- **不出現在 Alt+Tab 清單**：視窗是 tool window
- **總是最上層**（預設）：可在設定中關閉
- **可拖曳**：用手動捕捉滑鼠實作（因 NoActivate 下 `DragMove` 不穩定）
- **6 種尺寸**：微巧 / 小巧 / 輕巧 / 標準 / 加大 / 特大，迷你系列沒有關聯字預覽

## 十、常見問題

**Q: 啟用了「文字鍵盤左手」後，記事本打不出字母了？**
A: 正確，字母鍵被攔截作為九宮格輸入。按 `numpad /` 隱藏視窗，或從導覽列關掉該裝置即可。

**Q: 不小心把視窗拖到螢幕外，找不到了？**
A: 右鍵托盤圖示 → 重設位置，或按 `numpad /` 先隱藏再顯示（預設出現在上次位置的附近）。

**Q: 按了字碼但沒有候選字跳出？**
A: 可能是 DEMO 版字庫沒有收錄這個字。DEMO 字庫刻意縮減，完整版需自行補齊或等待正式版。

**Q: 可以變更某個筆型的按鍵位置嗎？**
A: 可以，但要同步替換字碼表和 UI 圖片。詳見
[`02_可替換資料總覽.md`](02_可替換資料總覽.md)。
