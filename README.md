# 九格輸入法 G9input — DEMO 版

九格輸入法是一個在 Windows 桌面常駐、對任何有焦點的文字欄位模擬鍵盤送字的中文輸入工具。
它**不掛在作業系統的 IME 框架裡**，而是以一個可隨時隱藏／喚醒的小浮動視窗形式運作。

輸入方式：把數字鍵盤的 `1`-`9` 當成九宮格，每個格子對應一種**基本筆型**。把一個中文字
視覺上拆成 1-3 個主要筆劃，依序按對應的數字鍵即可查字。

```
7 8 9
4 5 6      ← 九宮格對應 numpad 1~9
1 2 3
```

---

## 特色與功能

### 核心輸入
- **九宮拆字**：最多打 3 碼就看到 9 個候選字，適合高速盲打
- **標點模式**：按 `0` 進入 9 個常用標點；再按「下一頁」可切到擴充標點（含全形符號、希臘字母、製表符等約 370 字元）
- **百家姓**：先按首碼再按 `0`，顯示該首碼底下的姓氏清單，人名輸入更快
- **關聯字預覽**：送出一字後自動顯示下一字的 9 個候選（標準尺寸），不必再打碼
- **右鍵粵語同音字**：在選字狀態下右鍵任一格,跳出該字的粵語同音字群
- **多頁翻頁**:候選字超過 9 個時可用 `+` / `-` 翻頁

### 5 種輸入裝置（可單用或組合）

| 裝置 | 說明 |
|---|---|
| A. NumLock 開的 numpad | Windows 全域熱鍵，需 NumLock 開 |
| B. NumLock 關的 numpad | 低階鉤子攔截掃描碼，不論 NumLock 狀態 |
| C. 文字鍵盤數字列 | `1234567890-=` 這排 |
| D. 文字鍵盤左手 | QWE/ASD/ZXC/VB 對應 789/456/123/0. |
| E. 文字鍵盤右手 | YUI/HJK/BNM/,. 對應 789/456/123/0. |

D 和 E 在 `B` 鍵衝突，強制互斥。

### 4 種輸出模式

| 模式 | 原理 | 適用情境 |
|---|---|---|
| 剪貼簿 | Ctrl+V | 最相容，適合老舊應用 |
| 直出（預設） | `KEYEVENTF_UNICODE` | 現代應用最佳選擇 |
| 轉倉 | 查字對倉頡碼，送 VK 事件給系統倉頡 IME | 只認特定 IME 的表單 |
| 轉易 | 查字對大易碼，送 VK 事件給系統大易 IME | 同上 |

### 顯示與相容
- **6 種視窗尺寸**：微巧、小巧、輕巧、標準、加大、特大（迷你系列不顯示關聯字預覽）
- **HKSCS PUA 字元自動轉換**：輸出時自動把 PUA 區的 HKSCS 字換成標準 Unicode，在不支援 HKSCS 的應用裡也能正常顯示。候選字 UI 上以**紅色**標示「會被轉換」的字
- **繁 ↔ 簡 輸出切換**：UI 永遠顯示繁體，送字時依設定決定是否轉簡
- **編碼過濾**：Han 模式會依繁簡當下設定以 Big5 或 GB2312 過濾，避免送出目標編碼不支援的字
- **視窗不搶焦點**：點視窗本體不會把焦點從文字欄位搶走
- **不出現在 Alt+Tab**：tool window 形式

### 全域熱鍵

| 熱鍵 | 功能 |
|---|---|
| numpad `/` | 顯示 / 隱藏輸入法視窗 |
| ScrollLock（若啟用替代模式） | 同上，避免與其他程式的 `RegisterHotKey` 衝突 |
| numpad `*` | 循環對齊視窗到螢幕四角 |

### 可攜（USB）模式
可在托盤選單開啟。啟用後程式會在 exe 同目錄寫入 `settings.conf`（簡單的 `key=value` 純文字），
把整個資料夾放到 USB 隨身碟，設定跟著走，關閉時會自動刪除。

## 快速開始

1. 從 Releases 下載最新 `G9inputDemo.exe` 並放到任意資料夾（例如 `D:\Portable\G9input\`）
2. 雙擊啟動。右下角出現綠色鑽石托盤圖示，左上角出現輸入法視窗
3. 切到記事本等文字欄位，確認游標在文字框裡
4. 按 numpad `1`~`9` 開始拆字（先看最左上的筆劃、再看最右上的筆劃、最後看末筆）
5. 打到足夠碼數時候選字跳出，再按 `1`~`9` 選字

要替換字庫或 UI：把本文件附帶的 `sample/Data/` 或 `sample/UI/` 複製到 exe 同目錄、
分別改名為 `Data\` 和 `UI\`，修改其中檔案後重啟程式即可。詳見
[`02_可替換資料總覽.md`](02_可替換資料總覽.md)。

## 隨附 Sample

本文件附帶一份**精簡版的格式範例**，位於 `sample/` 資料夾：

```
sample/
├── Data/                每個 txt 只收錄幾行作為格式示範
│   ├── char_map.txt     九宮格字碼表（示範各類 key：標點、擴充、百家姓、3 碼字、* 佔位）
│   ├── rel_map.txt      關聯字表（示範幾個常用字的下一字預測）
│   ├── cj5_map.txt      倉頡碼表
│   ├── b5_map.txt       大易碼表
│   ├── homo_map.txt     粵語同音字表（含 0~9 數字音群與漢字 key 兩種範例）
│   ├── b5_uni.txt       Big5↔Unicode 對照（HKSCS PUA 轉換）
│   └── t2s.txt          繁簡轉換表
└── UI/
    ├── 0s.png, 0n.png, 0c.png, 0f.png   4 張狀態背景圖，統一 204×206（含格線與字區框）
    ├── 1.png ~ 9.png                    9 張首碼提示圖，204×206（同上）
    └── 0-1.png ~ 9-9.png                90 張迷你 UI 單格圖，30×30 白底不透明
```

這份 sample 的定位是**格式範本**而非完整字庫：

- **Data/** 的每個 txt 只收錄數行作為格式示範——編碼、分隔符、特殊字元用法都正確，
  你可以照格式擴充成完整字庫
- **UI/** 的 normal UI 圖已畫上 3×3 格線和每格的字區建議框，當作繪製新圖時的對位底圖
- **UI/** 的 mini UI 小圖是空白透明 PNG，保留檔名與尺寸，你畫好內容覆蓋即可

使用方式：

- **當作格式起點**：複製需要的檔案到 exe 同目錄的 `Data\` 或 `UI\`，修改後重啟程式
- **對位繪圖**：把 `sample/UI/0s.png` 等圖匯入你的繪圖軟體當底圖，筆型就落在字區框裡
- **逐檔覆蓋**：只替換你想改的那幾個檔，其他仍使用 exe 內嵌的預設完整資源
